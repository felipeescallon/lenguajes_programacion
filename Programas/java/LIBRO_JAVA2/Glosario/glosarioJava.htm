Glosario de Java y Términos Relacionados

100% Pure Java(TM)
Java puro
Iniciativa de Sun Microsystems para guiar a los desarrolladores al escribir, certificar y comercializar aplicaciones.
A

Abstract Window Toolkit (AWT)
Kit de herramientas de ventanas abstractas (AWT)
Grupo de componentes de interfaz gráfica de usuario que se han implementado utilizando versiones de la plataforma nativa de los componentes. Estos componentes proporcionan el subconjunto de funciones que es común a todas las plataformas nativas. Sustituido en gran medida por el conjunto de componentes Swing. Véase también conjunto Swing.

abstract class
clase abstracta
Clase que contiene uno o varios métodos abstractos y de la que, por tanto, no pueden crearse instancias. Se definen las clases abstractas para que el resto de clases puedan ampliarlas y hacerlas concretas implementando los métodos abstractos.

abstract method
método abstracto
Método que no tiene ninguna implementación.

actual parameter list
lista de parámetros reales
Argumentos especificados en la llamada a un método determinado. Véase también lista de parámetros formales.

alpha value
valor alfa
Valor que indica la opacidad de un píxel.

API
interfaz para la programación de aplicaciones (API)
Interfaz para la programación de aplicaciones. Especificación que determina la forma en que un programador que está escribiendo una aplicación accede al comportamiento y estado de clases y objetos.

appliances
accesorios
Dispositivos conectados a una red, como por ejemplo impresoras, terminales Java y clientes que se gestionan mediante aplicaciones construidas utilizando JMAPI.

applet
applet
Programa escrito en Java y que se ejecuta en un navegador compatible con Java como por ejemplo HotJava(TM) o Netscape Navigator(TM).

argument
argumento
Elemento de datos especificado en una llmada de método. Un argumento puede ser un valor literal, una variable o una expresión.

array
matriz
Conjunto de elementos de datos de igual tipo que se distinguen por un número entero.

ASCII
ASCII
American Standard Code for Information Interchange. Asignación numérica de códigos numéricos de siete bits a caracteres. Véase también Unicode.

atomic
atómica
Se refiere a una operación que nunca se interrumpe ni se deja en estado incompleto bajo ninguna circunstancia.

B

Bean
Bean
Componente de software reutilizable. Estos componentes pueden combinarse para crear una aplicación.

binary operator
operador binario
Operador con dos argumentos.

bit
bit
Unidad de información más pequeña de un computador con un valor de 0 o 1.

bitwise operator
operador a nivel de bit
Operador que manipula dos valores comparando cada bit de un valor con el bit correspondiente del otro.

block
bloque
En Java, todo el código que esté entre llaves concordantes. Ejemplo: { x = 1; }.

boolean
Booleano
Se refiere a una expresión o variable que sólo puede tener un valor verdadero o falso. Java proporciona el tipo booleano y los valores literales verdadero o falso.

bounding box
cuadro delimitador
Para un objeto de barrido, es el rectángulo más pequeño que contiene todos los píxeles que no son completamente transparentes.

byte
byte
Secuencia de ocho bits. Java proporciona un tipo de byte correspondiente.

bytecode
bytecode
Código independiente de la máquina generado por el compilador de Java y ejecutado por el intérprete de Java.

C

casting
intercalar
Conversión explícita de un tipo de datos a otro.

class
clase
En Java, método que se llama sin hacer referencia a un objeto determinado. Una definición de clase define una instancia, métodos y variables de clase. También especifica las interfaces que la clase implementa y la superclase inmediata de dicha clase. Si la superclase no se especifica, la superclase será Object.

class method
método de clase
Método que se llama sin hacer referencia a un objeto determinado. Los métodos de clase afectan a toda la clase, y no a una instancia determinada de ésta. También denominado método estático . Véase también método de instancia.

classpath
CLASSPATH
CLASPATH es una variable de entorno que indica a la máquina virtual Java y a otras aplicaciones Java (por ejemplo, las herramientas Java ubicadas en el directorio JDK1.1.X\bin) dónde encontrar las bibliotecas de clase, incluidas las bibliotecas de clase definidas por el usuario.

class variable
variable de clase
Elemento de datos asociado con una clase determinada como un conjunto, y no con instancias determinadas de la clase. Las variables de clase se definen en las definiciones de clase. También denominado campo estático. Véase también variable de instancia.

client
cliente
En el modelo de comunicaciones cliente/servidor, el cliente es un proceso que permite acceder de forma remota a los recursos del servidor de un computador, tales como potencia de computación y gran capacidad de memoria.

codebase
base de códigos
Funciona en conjunción con el atributo código de la etiqueta <APPLET> para proporcionar una especificación completa sobre dónde hallar el archivo de clase applet principal: código especifica el nombre del archivo y base de códigos especifica el URL del directorio que contiene dicho archivo.

comment
comentario
En un programa, es el texto explicativo ignorado por el compilador. En los programas de Java los programas están delimitados utilizando // o /*...*/.

compilation unit
unidad de compilación
Unidad de código de origen más pequeña que puede compilarse. En la implementación actual de Java la unidad de compilación es un archivo.

compiler
compilador
Programa que traduce código de origen en código para que lo ejecute un computador. El compilador de Java traduce código de origen de Java en bytecode de Java. Véase también intérprete.

compositing
composición
Proceso de superponer una imagen sobre otra para crear una sola imagen.

constructor
constructor
Método que crea un objeto. En Java, los constructores son métodos de instancia con el mismo nombre que su clase. Los constructores Java se invocan con el código new.

core class
clase central
Clase pública (o interfaz) que es el miembro estándar de la plataforma Java. Se trata de que, al menos, las clases centrales de Java estén disponibles en todos los sistemas operativos dónde se ejecuta la plataforma Java. Un programa totalmente Java al 100% sólo se apoya en clases centrales; esto quiere decir que puede ejecutarse en cualquier parte. Todas las clases centrales residen en el paquete java.*.

critical section
sección crítica
Segmento de código en el que un subproceso utiliza recursos (como determinadas variables de instancia) que pueden utilizarse por otros subprocesos, pero que no se pueden utilizar al mismo tiempo.

D

declaration
declaración
Instrucción que establece un identificador y le asocia atributos sin reservar el espacio de almacenamiento (para datos) o proporcionar la implementación (para métodos). Véase también definición.

definition
definición
Declaración que reserva espacio de almacenamiento (para datos) o proporciona implementación (para métodos). Véase también declaración.

deprecation
desprecio
Se refiere a una clase, interfaz, constructor, método o campo que ya no se recomienda y que puede desaparecer en una versión posterior.

derived from
derivada de
La clase X está "derivada de" la clase Y si la clase X extiende la clase Y. Véase también subclase, superclase.

distributed
distribuido
Que se ejecuta en más de un espacio de direcciones.

double precision
doble precisión
En la especificación del lenguaje Java, describe un número de coma flotante que contiene 64 bits de datos. Véase también precisión sencilla.

E

EmbeddedJava(TM)
Java intercalado
Entorno de sistema de runtime de Java para los dispositivos intercalados de gran volumen.

encapsulation
encapsulado
Ubicación de conocimiento dentro de un módulo. Debido a que los objetos encapsulan datos e implementaciones, el usuario de un objeto puede verlo como un cuadro negro que proporciona servicios. Las variables de instancias y métodos pueden añadirse, eliminarse o modificarse. Sin embargo, mientras los servicios que el objeto suministran sean idénticos, el código que utiliza el objeto puede seguir utilizándolo sin necesidad de volver a escribirlo. Véase también variable de instancia, método de instancia.

exception
excepción
Suceso que se produce durante la ejecución de un programa y que impide que el programa continúe funcionando correctamente, normalmente conocido como error. Java soporta excepciones con las palabras clave try, catch y throw. Véase también manejador de excepciones.

exception handler
manejador de excepciones
Bloque de código que reacciona a un tipo de excepción específico. Si se trata de una excepción de un error del que puede recuperarse el programa, éste puede continuar su ejecución una vez ejecutado el manejador de excepciones.

executable content
contenido ejecutable
Aplicación que se ejecuta desde dentro de un archivo HTML. Véase también applet.

extends
amplía
La clase X amplía la clase Y para agregar funciones agregando campos o métodos a la clase Y o sustituyendo métodos de la clase Y. Una interfaz amplía otra interfaz agregando métodos. Se dice que la clase X es una subclase de la clase Y. Véase también derivado de.

F

FCS
primera entrega al cliente (FCS)
Primera entrega al cliente. Día en el que un producto se comercializa/envía al cliente.

field
campo
Miembro de datos de una clase. Si no se especifica, un campo no suele ser estático

FTP
FTP
Protocolo de transferencia de archivos básico (FTP). Este protocolo, basado en TCP/IP, permite activar la recogida y almacenamiento de archivos entre sistemas centrales de Internet. Véase también TCP/IP.

formal parameter list
lista de parámetros formal
Parámetros especificados en la definición de un método determinado. Véase también lista de parámetros reales.

G

garbage collection
recogida de residuos
Detección y liberación automática de memoria que ya no está en uso. El sistema en runtime de Java realiza la recogida de residuos, de esta forma los programadores nunca liberan los objetos de forma específica.

GUI
interfaz gráfica de usuario (GUI)
Interfaz gráfica de usuario. Indica las técnicas que comprenden la utilización de gráficos, teclado y ratón para ofrecer al usuario una interfaz fácil de utilizar para un programa determinado.

H

hexadecimal
hexadecimal
Sistema de numeración que utiliza 16 como base. Las señales 0-9 y a-f (o A-F) representan los dígitos 0 a 15. En programas Java, los números hexadecimales deben ir precedidos de 0x. Véase también octal.

hierarchy
jerarquía
Clasificación de relaciones en la que cada elemento es una forma especializada de su elemento superior, excepto el de nivel superior (conocido como raíz). Cada elemento puede tener uno o varios elementos debajo suyo en la jerarquía. En la jerarquía de clases de Java, la raíz es la clase Object.

HotJava(TM) Browser
Navegador HotJava (TM)
Navegador de web que puede personalizarse fácilmente desarrollado por Sun Microsystems y escrito en Java.

HTML
HTML
Lenguaje de marcas de hipertexto. Se trata de un formato de archivo, basado en SGML, para documentos de hipertexto que aparecen en Internet. Es muy simple y permite incorporar imágenes, sonido, vídeo, campos de formularios y texto simple. Las referencias a otros objetos están incorporadas utilizando referencias URL.

HTTP
HTTP
Protocolo de transferencia de hipertexto. Protocolo de Internet, basado en TCP/IP y utilizado para recoger objetos de hipertexto desde sistemas centrales remotos. Véase también TCP/IP.

I

IDL
IDL
Lenguaje de definición de interfaz Java. Aplicaciones API de Java que permiten conectividad e interoperabilidad basadas en estándares con CORBA (Common Object Request Broker Architecture).

identifier
identificador
Nombre de un elemento en un programa Java.

inheritance
herencia
Concepto de clases que contienen automáticamente las variables y los métodos definidos en sus supertipos. Véase también subclase, superclase.

instance
instancia
Objeto de una clase determinada. En los programas creados con Java, una instancia de una clase se crea con el operador new seguido por el nombre de la clase.

instance method
método de instancia
Cualquier método que se invoca respecto a una instancia de una clase. También se conoce como método. Véase también método de clase.

instance variable
variable de instancia
Cualquier elemento de datos que se asocia con un objeto determinado. Cada instancia de una clase tiene su propia copia de las variables de instancia definidas en la clase. También se conoce como campo. Véase también variable de clase.

interface
interfaz
En Java, se trata de un grupo de métodos que varias clases pueden implementar, independientemente de la ubicación de las las clases en la jerarquía de clases.

Internet
Internet
Red enorme que está formada por millones de sistemas centrales de diferentes organizaciones y países de todo el mundo. Esta red se conecta físicamente gracias a redes más pequeñas y transmisión de datos mediante un conjunto de protocolos comunes.

IP
IP
Protocolo de Internet. Protocolo básico de Internet. Permite la transmisión (no fiable) de paquetes individuales desde un sistema central a otro. No garantiza si el paquete se entregará o no, cuánto tardará, o si los diversos paquetes enviados se entregarán en el orden en que se han enviado. Los protocolos construidos sobre esto añaden las nociones de conexión y fiabilidad. Véase también TCP/IP.

interpreter
intérprete
Módulo que descodifica y ejecuta todas las instrucciones del código. El intérprete de Java descodifica y ejecuta bytecode de Java. Véase también compilador, sistema de runtime.

J

JAE
JAE
Entorno de la aplicación Java (Java Application Environment). Código de origen del Kit de desarrollo Java(TM).

JAR Files (.jar)
archivos JAR
Archivo de Java. Formato de archivo utilizado para agregar muchos archivos a uno.

JAR file format
formato de archivo JAR
JAR (archivo Java) es un formato de archivo independiente de plataforma que agrega muchos archivos en uno solo. Los applets múltiples de Java y sus componentes (archivos de clase, imágenes, sonidos y otros archivos de recursos) pueden reunirse en un archivo JAR y descargarse en un navegador en una sola transacción HTTP. También soporta la compresión de archivos y las firmas digitales.

Java(TM)
Java (TM)
Lenguaje de programación orientado a objetos y desarrollado por Sun Microsystems. Lenguaje de programación orientado a "escribir una vez, ejecutar en cualquier parte".

Java Application Environment (JAE)
entorno de aplicación JAVA (JAE)
Código de origen del Kit de desarrollo Java(TM).

JavaBeans(TM)
JavaBeans
Modelo de componente reutilizable y compatible con todas las plataformas.

Java(TM) Blend(TM)
Java(TM) Blend(TM)
Producto que permite a los desarrolladores simplificar el desarrollo de aplicaciones de base de datos mediante la asignación de registros a objetos Java y objetos Java a bases de datos.

Java(TM) Card(TM) API
Java(TM) Card(TM) API
Entorno de aplicación que cumple las normas ISO 7816-4 centrado en tarjetas inteligentes.

JavaCheck(TM)
JavaCheck(TM)
Herramienta que permite comprobar si los applets y aplicaciones cumplen ciertas especificaciones.

JavaChip(TM)
JavaChip (TM)
Procesador de Sun que ejecuta bytecode de Java de forma nativa. Con un chip de Java, bytecode ignora la máquina virtual o la etapa inmediata del compilador para pasar directamente al procesador.

Java Computing
computación Java
Arquitectura de computación que utiliza protocolos de red estándar que explota la disponibilidad de redes, datos y aplicaciones Java para ofrecer servicios dinámicos a una gran variedad de dispositivos activados por Java.

Java Database Connectivity (JDBC(TM))
conectividad de base de datos de Java (JDBC(TM))
Estándar reconocido en la industria para llevar a cabo conexiones independientes de bases de datos entre Java y una gran variedad de bases de datos. La conectividad JDBC(TM) proporciona una aplicación API para acceso a bases de datos basadas en API para SQL.

Java(SM) Developer Connection(SM) (JDC)
Java(SM) Developer Connection(SM) (JDC)
Servicio creado para desarrolladores individuales. Este servicio ofrece formación en línea, descuentos en productos, artículos especializados, información sobre errores y temprano acceso a funciones.

Java(TM) Development Kit (JDK(TM))
Kit de desarrollo Java (TM) (JDK(TM))
Entorno de desarrollo de software para escribir applets en Java.

Java Electronic Commerce Framework (JECF)
Java Electronic Commerce Framework (JECF)
Arquitectura estructurada para el desarrollo de aplicaciones destinadas al desarrollo de aplicaciones comerciales electrónicas en Java.

Java Enterprise API
API Enterprise de Java
Esta aplicación API facilita el trabajo de creación de aplicaciones para bases de datos y para transacciones comerciales a gran escala que pueden compartir datos multimedia con otras aplicaciones dentro de una organización o en Internet. Se han creado cuatro aplicaciones API dentro de la familia API Enterprise de Java.

Java(TM) Foundation Class (JFC)
Java(TM) Foundation Class (JFC)
Extensión que añade bibliotecas de clase de interfaz gráfica de usuario al kit de herramientas de ventanas abstractas, Abstract Windowing Toolkit (AWT).

Java Interface Definition Language (IDL)
lenguaje de definición de interfaz de Java (IDL)
Aplicaciones API de Java que ofrecen conectividad e interoperabilidad basada en estándares con CORBA (Common Object Request Broker Architecture).

Java Media APIs
API Media de Java
Conjunto de aplicaciones API que soporta la integración de clips de vídeo y de audio, fuentes en 2D, gráficos, imágenes y telefonía.

Java Media Framework (JMF)
Java Media Framework (JMF)
La estructura central soporta relojes para llevar a cabo sincronizaciones entre diferentes tipos de soportes (por ej., salida de audio y de vídeo). La estructura de extensión estándar permite a los usuarios realizar un flujo completo de audio y de vídeo.

Java(TM) Naming and Directory Interface(TM) (JNDI)
Java(TM) Naming and Directory Interface(TM) (JNDI)
Conjunto de aplicaciones API que actúan como interfaz de múltiples servicios de directorio y de nombres.

JavaOS(TM)
JavaOS(TM)
Plataforma optimizada para ejecutar Java en una gran variedad de plataformas informáticas y para consumidor. JavaOS proporciona un runtime optimizado para ejecutar aplicaciones Java directamente en plataformas de hardware sin necesidad de un sistema operativo central.

JavaPlan(TM)
JavaPlan(TM)
Herramienta de diseño y creación de diagramas orientada a objetos y escrita en Java.

Java Platform
Plataforma Java
La máquina virtual Java(TM) y el núcleo central de las clases de Java constituyen la Plataforma Java. La Plataforma Java proporciona una interfaz de programación uniforme para un programa 100% Java, independientemente del sistema operativo subyacente.

Java Remote Method Invocation (RMI)
llamada a método remoto (RMI)
Modelo de objeto distribuido para aplicaciones Java a Java, en el que los métodos remotos de objetos Java pueden ser invocados desde las máquinas virtuales Java, y probablemente desde sistemas distintos.

Java Runtime Environment (JRE)
entorno de runtime de Java (JRE)
Subconjunto del Kit de desarrollo Java(TM) para usuarios y desarrolladores que desean volver a distribuir el JRE. El JRE está formado por la máquina virtual Java, las clases de núcleo central Java y los archivos de soporte.

JavaSafe(TM)
JavaSafe(TM)
Herramienta que permite gestionar y hacer un seguimiento de los cambios realizados en archivos de origen. Esta herramienta está escrita en Java.

JavaScript(TM)
JavaScript(TM)
Lenguaje de instrucciones Web que los servidores Web y navegadores utilizan. El nombre parece indicar que está relacionado con Java y a veces causa confusión. Como cualquier lenguaje de instrucciones, se utiliza fundamentalmente para unir otros componentes o aceptar entradas del usuario.

Java(TM) Studio(TM)
Java(TM) Studio(TM)
Primer programa Java que permite crear aplicaciones y applets con facilidad y sin necesidad de conocer Java.

Java(TM) Virtual Machine (JVM)
máquina virtual Java(TM) (JVM)
Parte del entorno runtime de Java que es responsable de interpretar los bytecodes de Java.

Java(TM) Web Server(TM)
Java(TM) Web Server(TM)
Solución de plataforma independiente, segura, fácil de administrar, ampliable y fácil de utilizar que contribuye a acelerar y simplificar la gestión de las redes Web de Intranet e Internet. Proporciona una productividad inmediata para las aplicaciones de servidor Java con funciones completas.

Java(TM) Workshop(TM)
Java(TM) Workshop(TM)
Conjunto completo de herramientas integradas en un entorno simple para gestionar la programación de Java. Java Workshop utiliza una estructura modular que permite conectar nuevas herramientas a la estructura general.

Java Wallet
Java Wallet
Interfaz de usuario, construida en el marco de comercio electrónico Java (Java Electronic Commerce Framework), que permite realizar compras en línea, transferencias de valores y funciones administrativas.

JavaSpaces(TM)
JavaSpaces(TM)
Proporciona mecanismos de intercambio de datos y de distribución para código Java.

JavaSoft(TM)
JavaSoft(TM)
Unidad comercial antigua de Sun Microsystems, Inc., ahora conocida como Sun Microsystems, Inc., Java Software Division.

JDBC(TM)
JDBC(TM)
Conectividad de base de datos de Java. Estándar reconocido en la industria para llevar a cabo conexiones independientes de bases de datos entre Java y una gran variedad de bases de datos. La conectividad JDBC(TM) proporciona una aplicación API para acceso a bases de datos basadas en API para SQL.

JDC
JDC
Java(SM) Developer Connection(SM). Servicio creado para desarrolladores independientes que proporciona formación en línea, descuento en productos, información sobre errores y acceso inmediato a funciones y aplicaciones.

JDK(TM)
JDK
Kit de desarrollo Java(TM). Entorno de desarrollo de software para escribir applets y aplicaciones en Java.

JECF
JECF
Red de comercio electrónico en Java (Java Electronic Commerce Framework). Arquitectura estructurada que permite el desarrollo de aplicaciones orientadas al comercio electrónico en Java.

JFC
JFC
Clase básica en Java(TM). Extensión que añade bibliotecas de clase de interfaz gráfica de usuario a Abstract Windowing Toolkit (AWT).

JMAPI
JMAPI
Java Management API. Conjunto de clases e interfaces sobre el lenguaje Java que permite a los desarrolladores construir aplicaciones de sistema, red y gestión de servicios.

JNDI
JNDI
Java(TM) Naming and Directory Interface(TM). Conjunto de aplicaciones API que ayudan en la interfaz entre diversos servicios de directorios y de nombrado.

JPEG
JPEG
Grupo de expertos en fotografía unidos (JPEG). Estándar de compresión de archivos de imágenes creado por este grupo. Este estándar consigue un nivel de compresión muy alto, gracias a la introducción de distorsiones en la imagen que apenas se perciben.

JRE
JRE
Entorno runtime de Java. Subconjunto del Kit de desarrollo Java para usuarios y desarrolladores que desean redistribuir JRE. Este entorno JRE está constituido por la máquina virtual Java, las clases centrales de Java y los archivos de soporte.

Just-in-time (JIT) Compiler
compilador Just-in-time (JIT)
Compilador que convierte todo el bytecode en código de la máquina nativa, de la misma forma que se ejecuta un programa Java. Como consecuencia, se mejora la velocidad en runtime en comparación con el código interpretado por una máquina virtual Java.

JVM
JVM
Máquina virtual Java. Parte del entorno runtime de Java que es responsable de interpretar los bytecodes de Java.

K

L

lexical
léxico
Relacionado con cómo se traducen los caracteres del código de origen a símbolos que puede comprender el compilador.

linker
enlazador
Programa que acepta los archivos de código objeto compilados y genera un archivo ejecutable. El enlazador de Java crear un programa ejecutable desde clases compiladas. Véase también compilador, intérprete, sistema de runtime.

literal
literal
Representación básica de cualquier entero, coma flotante o valor de carácter. Por ejemplo, 3,0 es una coma flotante de precisión única literal y "a" es un carácter literal.

local variable
variable local
Elemento de datos conocido dentro de un bloque pero inaccesible a código fuera del bloque. Por ejemplo, cualquier variable definida dentro de un método Java es una variable local y no puede utilizarse fuera de dicho método.

M

member
miembro
Campo o método de una clase. A menos que se especifique lo contrario, los miembros no son estáticos.

method
método
Función definida en una clase. Véase también método de instancia, método de clase. A menos que se especifique lo contrario, los métodos no son estáticos.

Mosaic
Mosaic
Programa que proporciona una GUI simple que permite acceder fácilmente a los datos almacenados en Internet. Estos datos pueden ser archivos simples o documentos con hipertexto. Mosaic ha sido escrito por un equipo en NCSA.

multithreaded
multisubproceso
Describe un programa que se ha creado para que distintas partes del código se ejecuten simultáneamente. Véase también subproceso.

N

NCSA
NCSA
Centro nacional para aplicaciones de supercomputación (National Center for Supercomputer Applications). Véase también Mosaic.

O

object
objeto
Bloque de construcción principal de los programas orientados a objetos. Cada objeto es una unidad de programación que está constituida por (variables de instancia) y (métodos de instancia funcionales). Véase también clase.

object-oriented design
diseño orientado a objetos
Método de diseño de software que modela las características de los objetos abstractos o reales mediante clases y objetos.

octal
octal
Sistema de representación numérica en base 8 que utiliza los numerales 0 a 7 como dígitos. En los programas Java, los números octales deben ir precedidos por 0. Véase también hexadecimal.

overloading
sobrecarga
Utilizar un identificador para hacer referencia a varios elementos del mismo ámbito. En Java puede sobrecargar métodos pero no variables u operadores.

overriding
ignorar

Sandbox
zona protegida
Comprende una serie de componentes de sistema en cooperación, desde gestores de seguridad que se ejecutan como parte de la aplicación a medidas de seguridad creadas en la máquina virtual Java y con el lenguaje Java. La zona protegida garantiza que una aplicación poco fiable y, probablemente, con malas intenciones, acceda a los recursos del sistema.

scope
ámbito
Característica de un identificador que determina dónde puede utilizarse dicho identificador. La mayoría de los identificadores de Java tienen un ámbito local o de clase. Los métodos y variables de instancia y de clase tienen ámbito de clase. Pueden utilizarse fuera de la clase y las susbclases con sólo añadir como prefijo una instancia de la clase o (para variables de clase y métodos) el nombre de clase. Las otras variables se declaran dentro de los métodos y tienen ámbito local; sólo pueden utilizarse dentro del bloque que las encierra.

URL
URL
Localizador de recursos uniforme. Estándar para escribir una referencia a texto en una serie de datos en la WWW. Un URL es parecido a "protocolo://sistema/información local" donde "protocolo" especifica un protocolo que se utiliza para recoger el objeto (como HTTP o FTP), "sistema" especifica el nombre del sistema en Internet en el que se realiza la búsqueda e "información local" es una cadena (normalmente el nombre de un archivo) que se transfiere al manejador del protocolo en el sistema remoto.

virtual machine
máquina virtual
Especificación abstracta para un dispositivo de computación que puede implementarse de varias formas por software o hardware. La forma de compilar un conjunto de instrucciones de una máquina virtual es muy similar a como se compila el conjunto de instrucciones de un microprocesador. La Máquina virtual de Java está constituida por un conjunto de instrucciones de bytecode, un conjunto de registros, una pila, un conjunto de residuos y un área para almacenar métodos.
